System Architecture & Components

Modular Ingestion Flow

The Smart Transaction Stack consists of a high-performance transaction pipeline designed to optimize bundle submission timing, calculate tip floors dynamically, and autonomously handle errors using AI-driven reasoning.

01

Live Slot Monitor

Subscribes to slot updates via Yellowstone gRPC, falling back to Web3 WebSockets. Emits live slots to coordinate ingestion and Jito leader slots.

→
02

AI Decision Engine

Decides submission timing based on Jito slot distance, evaluates network congestion to dynamically tip, and reasons about failures.

→
03

Jito Bundle Builder

Compiles a VersionedTransaction, signs it, and attaches a dynamic tip payment targeting one of the 8 Jito validator tip accounts.

→
04

Lifecycle Monitor

Tracks transaction status (Submitted → Processed → Confirmed → Finalized) in real-time, calculating latencies and classifying failures.

Failure Classification & AI Recovery

When a transaction fails, it is passed to the Failure Classifier to determine the root cause (e.g. EXPIRED_BLOCKHASH, FEE_TOO_LOW). The AI Agent then adjusts parameters (tip percentile and multipliers), refreshes the blockhash using confirmed commitment, and resubmits autonomously.

Triton Geyser gRPC Subscription

To ensure high throughput and low latency, we use Yellowstone gRPC. To prevent mainnet node connection flooding, we apply a specific accountInclude filter targeting only our sender wallet. This filters out all other network activity at the server level.
